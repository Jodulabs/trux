# Phase 6.5 — QR pairing

*Branch: `phase-6.5-qr-pairing` · Companion: [roadmap](../../../docs/2026-06-16-trux-roadmap.md)*

---

## Goal

Remove the one remaining phone-onboarding friction after Phase 5: **typing the bearer token**. The
desk shows a QR encoding the Tailscale URL **+** the token; the phone scans it and lands in trux
already authenticated — no typing.

Persona is settled: **build for me, over Tailscale.** So this is the *only* Phase 6.5 item — mDNS is
dropped (redundant with Tailscale MagicDNS) and Cloudflare Tunnel is deferred (no public door on
Tailscale). See [roadmap Phase 6.5](../../../docs/2026-06-16-trux-roadmap.md).

---

## Flow

```
Desk (any access: localhost or tailnet)        Phone (already on the tailnet)
  ┌──────────────────────────────┐
  │ Sidebar → "Pair phone"       │
  │ PairModal shows a QR:        │   scan with camera
  │  https://<host>.ts.net/      │ ───────────────────────▶ opens URL in browser
  │     #token=<bearer>          │                            main.tsx: consumePairingToken()
  └──────────────────────────────┘                              → localStorage['trux_token']=<bearer>
                                                                 → strip #fragment
                                                                 → App boots authenticated (no TokenGate)
```

**Precondition unchanged:** the phone must already have the Tailscale client and be on the tailnet
(that's the accepted one-time setup for this persona). QR removes the *token typing*, not the
Tailscale install.

**No backend change.** `GET /config` already returns `{ tailscaleHost }` (Phase 5); the QR is built
entirely client-side from `tailscaleHost` + `localStorage['trux_token']`.

## 2. Phone side — consume the token from the URL fragment

New `apps/frontend/src/pairing.ts` — a pure, testable helper:

```ts
// Reads a pairing token from the URL fragment, stores it, and strips the fragment.
// Fragment (not query) → the token is NEVER sent to the server or written to access logs.
export function consumePairingToken(loc: Location = window.location): string | null {
  const m = /[#&]token=([^&]+)/.exec(loc.hash)
  if (!m) return null
  const token = decodeURIComponent(m[1])
  localStorage.setItem('trux_token', token)
  history.replaceState(null, '', loc.pathname + loc.search) // drop the secret from the visible URL
  return token
}
```

Call it in `apps/frontend/src/main.tsx` **before** `createRoot`, so the token is in `localStorage`
before `api.ts` reads it and before `App.tsx` runs `tryLoad`:

```ts
import { consumePairingToken } from './pairing'

consumePairingToken()

createRoot(document.getElementById('root')!).render(/* … unchanged … */)
```

Result: a freshly-paired phone skips `TokenGate` entirely — `tryLoad` succeeds on first try.

## 3. Desk side — show the QR

New `apps/frontend/src/components/PairModal.tsx`:

```tsx
import React from 'react'
import { QRCodeSVG } from 'qrcode.react'
import { useStore } from '../store'

interface Props { onClose: () => void }

export function PairModal({ onClose }: Props): React.ReactElement {
  const tailscaleHost = useStore((s) => s.tailscaleHost)
  const token = localStorage.getItem('trux_token')
  const url = tailscaleHost && token
    ? `https://${tailscaleHost}/#token=${encodeURIComponent(token)}`
    : null

  return (
    <div className="pair-modal" data-testid="pair-modal">
      {url ? (
        <>
          <p>Scan with your phone (must be on the tailnet):</p>
          <div className="pair-qr" data-testid="pair-qr">
            <QRCodeSVG value={url} size={240} />
          </div>
          <p className="hint">Opens trux already signed in — no token typing.</p>
        </>
      ) : (
        <p data-testid="pair-unavailable">
          Set <code>TRUX_TAILSCALE_HOST</code> and a bearer token to pair a phone.
        </p>
      )}
      <button onClick={onClose}>Close</button>
    </div>
  )
}
```

Wire a trigger into `apps/frontend/src/components/Sidebar.tsx` — add local state + a footer button
(it's currently stateless; add `useState` + the `PairModal` import):

```tsx
import { useState } from 'react'
import { PairModal } from './PairModal'
// …
const [pairing, setPairing] = useState(false)
// …inside <aside>, after the <ul>:
  <button className="pair-button" data-testid="pair-open" onClick={() => setPairing(true)}>
    📱 Pair phone
  </button>
  {pairing && <PairModal onClose={() => setPairing(false)} />}
```

## Security

Consistent with [design → Auth & security](../../../docs/2026-06-16-trux-design.md#auth--security):

- **Fragment, not query.** `#token=` is never transmitted to the backend → never in server access
  logs (honors "no secret in URLs/logs"). Stripped immediately via `history.replaceState`.
- The token in the QR is the *same* secret already in the desk's `localStorage`; the QR is shown
  locally and scanned in physical proximity. Residual exposure is the phone's browser history for the
  instant before the strip — acceptable for a single-user tool.
- **Deferred hardening (not now):** a one-time, short-TTL pairing code exchanged for the bearer
  (keeps the real token out of the QR/history). The design spec already prefers "simpler than a
  ticket dance for a single user," so this stays deferred unless the threat model changes.

## Testing

Vitest + Testing Library (matches the existing 19 frontend tests):

- `apps/frontend/src/pairing.test.ts`
  - `#token=abc` → `localStorage['trux_token'] === 'abc'` and `location.hash` cleared.

  - `useStore.setState({ tailscaleHost: 'box.tail123.ts.net' })` + token in `localStorage` → renders
    `pair-qr`.

**Done when:** at the desk, click "Pair phone", scan the QR with a phone on the tailnet, and trux
opens already authenticated with no typed token.
