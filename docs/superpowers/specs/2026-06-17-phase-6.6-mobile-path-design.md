## Goal

Make **one** workflow actually work end to end and be testable: Claude running on the dev box,
driven from the phone over Tailscale. Phase 5 (remote) and 6.5 (QR) were implemented but never run
for real — a live walk-through found the path was broken at the first step.

---

## Bugs found by walking the path live (not on paper)

Running the backend and curling the real endpoints surfaced two blockers that unit tests missed
because nothing ever served the built frontend:

1. **Frontend was never served.** `server.ts` resolved the dist dir as `../../../frontend/dist`
   (→ `trux/frontend/dist`); the real build is `apps/frontend/dist`. Off by one `../`, so
   `existsSync` failed silently and `GET /` returned a JSON 404. The phone would get nothing.
   → Fixed to `../../frontend/dist`; added a startup log (`serving frontend from …` /
   `no frontend build at …`) so this can never fail silently again.
2. **SPA fallback 500'd.** `fastifyStatic` was registered with `decorateReply: false`, which removes
   `reply.sendFile` — so the not-found handler (`reply.sendFile('index.html')`) threw 500 on every
   deep link / reload. → Removed `decorateReply: false`.

## Start & connect ergonomics

The priority was "starting trux on the dev machine" + "connecting from the phone". Changes:

- **`index.ts` auto-loads `~/.trux/.env`** (then repo-local `.env`, which wins per-key). A foreground
  `pnpm start` now works without systemd.
- **Startup pairing QR.** When `TRUX_TAILSCALE_HOST` + `TRUX_SECRET` are set, the backend prints a
  terminal QR encoding `https://<host>/#token=<secret>` (via `qrcode-terminal`). Start → scan →
  the phone is in, signed in. Always prints the local URL too.
- **Root scripts:** `pnpm build` (frontend) and `pnpm start` (backend) — the two commands the runbook
  uses.
- **[RUNBOOK.md](../../../docs/RUNBOOK.md)** — exact steps for the local sanity check and the
  phone-over-Tailscale path, env reference, and troubleshooting.

## Verified locally (this pass)

Via a live backend + curl + a `ws` client:

- `GET /` → 200 HTML; `/manifest.json`, `/assets/*.js` → 200; SPA fallback `/x/y/z` → 200.
- Startup QR banner renders.
- REST auth gate: no token → 401, good token → 200.
- Conversation create → ok.
- WS auth: bad token → `error (unauthorized)`; good token → `hello`; unknown id → `error`.
